# 테이블오더 서비스 — 사용자 페르소나

---

## Persona 1: 디지털 친숙 고객 (Tech-Savvy Customer)

| 항목 | 내용 |
|---|---|
| 이름 | 김민준 |
| 나이 | 28세 |
| 직업 | IT 회사 직원 |
| 디지털 숙련도 | 높음 |
| 이용 패턴 | 동료들과 그룹 식사, 개인 식사 모두 빈번 |

**특성**:
- 스마트폰/태블릿 사용에 익숙하며 새로운 디지털 서비스를 빠르게 습득
- 빠른 주문 처리와 효율성을 중시
- 분할 계산 기능을 적극 활용 (동료 식사 시)
- 주문 상태를 수시로 확인하는 편

**목표**:
- 대기 없이 빠르게 주문하고 싶다
- 메뉴를 한눈에 파악하고 싶다
- 그룹 식사 시 분할 계산을 편리하게 하고 싶다

**불만 요소**:
- 느린 응답 속도
- 직관적이지 않은 UI
- 불필요한 단계가 많은 주문 프로세스

## Persona 2: 디지털 비친숙 고객 (Non-Tech Customer)

| 항목 | 내용 |
|---|---|
| 이름 | 박영숙 |
| 나이 | 58세 |
| 직업 | 자영업자 |
| 디지털 숙련도 | 낮음 |
| 이용 패턴 | 가족 모임, 지인 식사 |

**특성**:
- 태블릿 사용 경험이 적어 조작에 시간이 걸림
- 큰 글씨와 명확한 버튼을 선호
- 메뉴 이미지를 보고 선택하는 것을 선호
- 실수로 잘못 주문할까 걱정

**목표**:
- 복잡하지 않게 메뉴를 보고 주문하고 싶다
- 주문 전에 내용을 다시 확인하고 싶다
- 잘못 주문했을 때 쉽게 수정하고 싶다

**불만 요소**:
- 작은 글씨와 버튼
- 복잡한 화면 구성
- 실수 시 되돌리기 어려운 프로세스

## Persona 3: 매장 관리자 (Store Admin)

| 항목 | 내용 |
|---|---|
| 이름 | 이정호 |
| 나이 | 42세 |
| 직업 | 식당 점주 |
| 디지털 숙련도 | 중간 |
| 이용 패턴 | 매일 영업 시간 중 지속 사용 |

**특성**:
- 바쁜 영업 시간에 빠르게 주문을 확인하고 처리해야 함
- 여러 테이블의 주문을 동시에 관리
- 메뉴 변경(가격, 품절 등)을 수시로 반영
- 테이블 회전율 관리에 관심

**목표**:
- 신규 주문을 즉시 확인하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 추가/수정/삭제하고 싶다
- 테이블 이용 완료 처리를 빠르게 하고 싶다

**불만 요소**:
- 주문 알림 지연
- 복잡한 관리 화면
- 테이블 상태 파악이 어려운 레이아웃

## Persona 4: 슈퍼 관리자 (Super Admin)

| 항목 | 내용 |
|---|---|
| 이름 | 최서연 |
| 나이 | 35세 |
| 직업 | 프랜차이즈 본사 운영팀장 |
| 디지털 숙련도 | 높음 |
| 이용 패턴 | 주 2~3회 접속, 계정 관리 및 이력 확인 |

**특성**:
- 다수 매장의 관리자 계정을 생성/관리
- 보안과 감사 추적에 민감
- 계정 변경 이력을 주기적으로 검토
- 효율적인 일괄 관리를 선호

**목표**:
- 매장 관리자 계정을 안전하게 생성/삭제하고 싶다
- 모든 계정 변경 이력을 추적하고 싶다
- 매장별 관리자 현황을 파악하고 싶다

**불만 요소**:
- 감사 이력 누락
- 계정 관리 절차가 복잡한 시스템
- 이력 검색/필터링이 불편한 UI

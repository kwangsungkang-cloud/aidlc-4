# 테이블오더 서비스 요구사항 문서

## Intent Analysis Summary
- **User Request**: 디지털 테이블오더 서비스 플랫폼 구축
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: Multiple Components (고객용 UI, 관리자용 UI, 백엔드 API, 데이터베이스, 클라우드 스토리지)
- **Complexity Estimate**: Moderate-Complex
- **Requirements Depth**: Standard

---

## 1. 기술 스택

| 영역 | 기술 |
|---|---|
| 백엔드 | Spring Boot (Java/Kotlin) |
| 프론트엔드 (고객용) | React (TypeScript) + shadcn/ui - 별도 앱 |
| 프론트엔드 (관리자용) | React (TypeScript) + shadcn/ui - 별도 앱 |
| 데이터베이스 | MySQL |
| 이미지 스토리지 | AWS S3 |
| 배포 환경 | AWS |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 기반 |

## 2. 기능 요구사항 (Functional Requirements)

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리
- **설명**: 고객이 별도 로그인 없이 즉시 주문 가능하도록 자동 인증
- **상세**:
  - 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호)
  - 로그인 정보 로컬 저장 후 자동 로그인
  - 16시간 세션 유지

### FR-01-1: 초기 접속 안내 화면
- **설명**: 고객이 처음 접속했을 때 환영 메시지와 서비스 사용법을 안내
- **상세**:
  - 자동 로그인 성공 후 첫 화면으로 안내 화면 표시
  - 매장명, 테이블 번호 표시
  - 환영 메시지 (greeting message)
  - 서비스 사용법 간단 안내 (메뉴 탐색 → 장바구니 담기 → 주문하기 흐름 설명)
  - "주문 시작하기" 버튼 클릭 시 메뉴 화면으로 이동
  - 세션 내 최초 1회만 표시 (이후 접속 시에는 바로 메뉴 화면)
- **UI 요구사항**:
  - 깔끔하고 직관적인 레이아웃
  - 디지털 비친숙 고객도 이해할 수 있는 큰 글씨와 아이콘
  - 터치 친화적 "주문 시작하기" 버튼

### FR-02: 메뉴 조회 및 탐색
- **설명**: 고객이 매장 메뉴를 카테고리별로 탐색
- **상세**:
  - 메뉴 화면이 기본 화면
  - 카테고리별 분류 및 빠른 이동
  - 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
  - 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

### FR-03: 장바구니 관리
- **설명**: 주문 전 메뉴 임시 저장 및 수정
- **상세**:
  - 메뉴 추가/삭제, 수량 조절
  - 총 금액 실시간 계산
  - 장바구니 비우기
  - 클라이언트 측 로컬 저장 (새로고침 시 유지)
  - 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- **설명**: 장바구니 메뉴를 실제 주문으로 전환
- **상세**:
  - 주문 내역 최종 확인 및 확정
  - 주문 성공 시: 주문 번호 표시, 장바구니 비우기, 메뉴 화면 자동 리다이렉트
  - 주문 실패 시: 에러 메시지 표시, 장바구니 유지
  - 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-04-1: 분할 계산
- **설명**: 테이블 세션의 총 주문 금액을 메뉴별 또는 인원별로 나눠서 계산 (화면 표시 전용, 실제 결제 미포함)
- **상세**:
  - 분할 시점: 계산 시점 (주문은 테이블 단위로 유지, 이용 완료 전 분할 계산 요청)
  - **메뉴별 분할**: 현재 세션의 전체 주문 메뉴 항목을 선택하여 별도 그룹으로 묶기 (예: 그룹A = 치킨+맥주, 그룹B = 피자+콜라), 그룹별 소계 표시
  - **인원별 분할**: 총 금액을 N명으로 균등 분할 (1/N), 인원 수 입력 후 1인당 금액 표시
  - 분할 결과는 화면에 표시만 함 (실제 결제는 매장에서 별도 처리)
  - 분할 계산은 서버에 저장하지 않음 (클라이언트 측 계산)
- **UI 요구사항**:
  - 주문 내역 화면에서 "분할 계산" 버튼 제공
  - 메뉴별/인원별 분할 모드 선택 탭
  - 메뉴별 분할: 드래그 또는 체크박스로 메뉴 항목을 그룹에 배정
  - 인원별 분할: 인원 수 입력 필드 + 1인당 금액 표시
  - 분할 결과 요약 화면

### FR-05: 주문 내역 조회 (고객)
- **설명**: 현재 테이블의 주문 이력 확인
- **상세**:
  - 주문 시간 순 정렬
  - 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
  - 현재 테이블 세션 주문만 표시
  - 매장 이용 완료 처리된 주문 제외

### FR-06: 매장 관리자 인증
- **설명**: 관리자가 매장 관리 시스템에 접근
- **상세**:
  - 매장 식별자 + 사용자명 + 비밀번호 입력
  - JWT 토큰 기반 인증, 12시간 세션 유지
  - 브라우저 새로고침 시 세션 유지
  - bcrypt 해싱, 로그인 시도 제한

### FR-07: 실시간 주문 모니터링
- **설명**: 들어오는 주문을 실시간으로 확인 및 관리
- **상세**:
  - SSE 기반 실시간 업데이트 (2초 이내)
  - 테이블별 그리드/카드 레이아웃
  - 각 테이블 카드: 총 주문액, 최신 주문 미리보기
  - 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
  - 주문 상태 변경 (대기중/준비중/완료)
  - 신규 주문 시각적 강조

### FR-08: 테이블 관리
- **설명**: 테이블별 주문 상태 및 세션 라이프사이클 관리
- **상세**:
  - 테이블 태블릿 초기 설정 (번호, 비밀번호, 세션 생성)
  - 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
  - 테이블 세션 종료 (이용 완료): 주문 내역 과거 이력 이동, 주문 목록/총액 리셋
  - 과거 주문 내역 조회: 시간 역순, 날짜 필터링

### FR-09: 메뉴 관리 (MVP 포함)
- **설명**: 메뉴 정보 동적 관리
- **상세**:
  - 메뉴 CRUD (등록/조회/수정/삭제)
  - 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
  - 이미지 S3 업로드 후 URL 자동 생성
  - 메뉴 노출 순서 조정
  - 필수 필드 및 가격 범위 검증

### FR-10: 슈퍼 관리자 계정 관리
- **설명**: 슈퍼 관리자가 매장 관리자 계정 생성/관리
- **상세**:
  - 매장 관리자 계정 생성/수정/삭제
  - 매장별 관리자 할당
  - 슈퍼 관리자 권한 분리

### FR-11: 슈퍼 관리자 활동 이력 관리
- **설명**: 슈퍼 관리자의 계정 생성/삭제 활동을 별도 이력으로 기록 및 조회
- **상세**:
  - 매장 관리자 계정 생성 시 이력 자동 기록 (수행자, 대상 계정, 매장, 시각, 액션 유형)
  - 매장 관리자 계정 삭제 시 이력 자동 기록 (수행자, 대상 계정, 매장, 시각, 액션 유형)
  - 이력 조회 화면: 시간 역순 정렬, 날짜 필터링, 액션 유형 필터링 (생성/삭제)
  - 이력 데이터는 삭제 불가 (감사 추적용, append-only)

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 주문 알림 2초 이내 전달
- 메뉴 조회 API 응답 시간 500ms 이내
- 동시 접속: 중규모 (10~50개 매장) 지원

### NFR-02: 보안
- JWT 토큰 기반 인증 (16시간 만료)
- bcrypt 비밀번호 해싱
- 로그인 시도 제한 (brute-force 방지)
- HTTPS 필수 (TLS 1.2+)
- SECURITY 확장 규칙 전체 적용 (SECURITY-01 ~ SECURITY-15)

### NFR-03: 가용성
- 매장 운영 시간 중 서비스 가용성 99.5% 이상
- 장바구니 데이터 클라이언트 측 유지 (서버 장애 시에도 보존)

### NFR-04: 확장성
- 멀티테넌트 구조 (매장별 데이터 격리)
- 10~50개 매장 동시 운영 지원
- 수평 확장 가능한 아키텍처

### NFR-06: 데이터 백업
- 일간 증분 백업 (매일 변경된 데이터만 백업)
- 주간 전체 백업 (매주 전체 데이터베이스 백업)
- 백업 데이터 보관 정책 수립
- 백업 복원 절차 문서화

### NFR-05: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 기반 레이아웃
- 고객용: 태블릿 최적화
- 관리자용: 데스크톱 웹 최적화

## 4. 제외 범위 (Out of Scope)

constraints.md에 정의된 대로 다음 기능은 구현하지 않음:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)

| 구성 요소 | 설명 |

## 6. 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보
- **Table**: 테이블 정보 (매장 소속)
- **TableSession**: 테이블 세션 (시작~이용완료)
- **Menu**: 메뉴 정보 (매장 소속, 카테고리별)
- **Category**: 메뉴 카테고리
- **Order**: 주문 정보 (테이블 세션 소속)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (이용 완료 후 이동)
- **Admin**: 관리자 계정 (매장 소속)
- **SuperAdmin**: 슈퍼 관리자 계정
- **AdminAuditLog**: 슈퍼 관리자 활동 이력 (수행자, 대상, 매장, 시각, 액션 유형)
